# Competitor Analysis Framework

## 분석 대상

- BECON
- ARAM Huvis
- ChoiceDX

## 분석 목적

### 문제
- 경쟁사를 각기 다른 기준으로 보면 브랜드, 웹사이트, SEO, UX, 제품, 세일즈, 신뢰 요소를 정확히 비교하기 어렵다.

### 원인
- 브랜드별 강점과 약점은 채널마다 다르게 나타나며, 검색노출과 AI Search, 글로벌 확장성까지 함께 봐야 한다.

### 해결
- 모든 브랜드를 동일한 항목과 동일한 SWOT 기준으로 분석한다.

### 실행
- 각 항목마다 Strength, Weakness, Opportunity, Threat를 작성한다.
- 관찰 근거가 있는 경우 URL, 화면, 검색 결과, 자료명을 함께 기록한다.
- ChoiceDX는 경쟁사 대비 개선 우선순위까지 연결한다.

### 성과
- BECON, ARAM Huvis, ChoiceDX의 경쟁 위치와 실행 우선순위를 한눈에 비교할 수 있다.

# 경쟁사 SWOT 분석

## Why
ChoiceDX를 성장시키려면 경쟁사의 노출 구조를 먼저 이해해야 한다. 이번 문서는 새로 받은 3개 사이트맵 파일을 기준으로 BECON, ARAM Huvis, ChoiceDX를 다시 분류하고, ChoiceDX가 어디에서 이길 수 있는지 정리한 것이다.

## What
분석 대상은 다음 3개 브랜드다.

- BECON
- ARAM Huvis
- ChoiceDX

## How
각 사이트맵 URL을 기준으로 아래 항목을 분류했다.

- 공식 도메인
- 외부 도메인
- 제품 페이지
- 서비스 페이지
- 사례/레퍼런스
- 블로그/콘텐츠
- SNS
- 앱/다운로드
- 쇼핑/커머스
- 글로벌
- 신뢰 요소
- URL 노이즈

## Result
ChoiceDX는 경쟁사보다 URL 수와 콘텐츠 양은 적지만, 구조가 단순하고 깨끗하다. 따라서 단순함을 약점이 아니라 강점으로 바꾸는 전략이 필요하다.
