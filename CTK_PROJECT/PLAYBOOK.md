# Reusable Brand Growth Playbook

## Why
브랜드 성장 프로젝트는 산업이 달라도 반복되는 구조가 있다. 회사와 제품은 달라져도, 고객이 브랜드를 발견하고 이해하고 비교하고 신뢰하고 문의하는 흐름은 대부분 같다.

## What
이 문서는 ChoiceDX 프로젝트에서 만든 운영 체계를 다른 사업분야에도 적용할 수 있도록 분리한 범용 템플릿이다.

## How
아래 항목은 프로젝트마다 교체하고, 운영 구조는 그대로 재사용한다.

| 유지할 구조 | 프로젝트마다 교체할 정보 |
|---|---|
| 분석 → IA → 전략 → 실행 → 검수 → 문서화 | 회사, 제품, 시장, 경쟁사, KPI |
| Competitor SWOT Framework | 경쟁사 목록, 사이트맵, 시장 기준 |
| SEO/GEO Audit | 키워드, 검색 결과, AI Search 결과 |
| UX Audit | 고객 여정, 앱/웹 구조, 화면 기준 |
| Task Backlog | 담당자, 일정, 우선순위 |
| Deliverables | 산출물 이름, 완료조건, 필요 자료 |
| Dashboard | 진행률, 이슈, 리스크, 의사결정 |
| Handover | 내부 운영 담당자, 반복 업무 |

## Result
새로운 사업분야에서도 빠르게 프로젝트 운영 구조를 만들고, 분석 기반으로 실행할 수 있다.

---

## 1. Project Setup

### 문제
프로젝트 시작 전에 맥락이 정리되지 않으면 실행 방향이 흔들린다.

### 원인
회사, 제품, 시장, 고객, 경쟁사, 법무, KPI 정보가 흩어져 있기 때문이다.

### 해결
작업 전 Project Context를 먼저 만든다.

### 실행
| 항목 | 확인 내용 |
|---|---|
| 회사 | 회사 소개, 사업 구조, 주요 고객, 내부 목표 |
| 제품 | 기능, 차별점, 가격/패키지, 도입 방식 |
| 고객 | 타깃 고객, 구매 의사결정자, 구매 여정 |
| 시장 | 시장 규모, 트렌드, 카테고리, 대체재 |
| 경쟁사 | 직접 경쟁사, 간접 경쟁사, 참고 브랜드 |
| 브랜드 | 메시지, 톤앤매너, 포지셔닝, 금지 표현 |
| 법무 | 산업별 규제, 광고 표현 제한, 개인정보 이슈 |
| KPI | 검색, 리드, 전환, 매출, 인수인계 기준 |
| 자료 | 웹사이트, 세일즈 자료, SNS, PR, Figma, 기존 문서 |

### 성과
모든 분석과 실행이 같은 맥락에서 진행된다.

## 2. Default Workflow

### 문제
실행이 분석보다 먼저 진행되면 재작업이 늘어난다.

### 원인
우선순위와 IA 없이 디자인, 카피, 채널 운영을 시작하기 때문이다.

### 해결
모든 작업은 고정된 순서로 진행한다.

### 실행
1. 현황 분석
2. 경쟁사 분석
3. 검색노출 분석
4. IA 설계
5. 우선순위 정의
6. 실행
7. 검수
8. 문서화
9. 다음 Action 제안

### 성과
프로젝트가 일회성 산출물이 아니라 반복 가능한 운영 체계로 남는다.

## 8. Dashboard Template

### 문제
프로젝트 상태가 흩어지면 PM이 다음 액션을 판단하기 어렵다.

### 원인
진행률, 이슈, 리스크, 결정사항, 막힌 항목이 따로 관리되기 때문이다.

### 해결
Dashboard를 주간 단위로 업데이트한다.

### 실행
| Section | 관리 내용 |
|---|---|
| Progress | 전체 진행률과 영역별 상태 |
| Issue | 현재 문제와 해결 상태 |
| Risk | 발생 가능 리스크와 대응책 |
| Decision | 결정 사항, 결정자, 후속 액션 |
| Done | 완료한 작업과 산출물 |
| Next Action | 다음 실행 항목 |
| Blocked | 막힌 항목과 필요한 입력값 |
| Priority | P0/P1/P2 우선순위 |
| Weekly Status | 주간 목표, 완료, 이슈, 다음 작업 |

### 성과
프로젝트의 현재 상태와 다음 행동이 한눈에 보인다.

## 9. Industry Adaptation Guide

### 문제
같은 템플릿을 모든 산업에 그대로 적용하면 중요한 산업별 기준이 빠질 수 있다.

### 원인
산업마다 구매 이유, 규제, 신뢰 요소, 전환 방식이 다르기 때문이다.

### 해결
기본 구조는 유지하되 산업별 체크포인트를 추가한다.

### 실행
| Industry | 추가로 봐야 할 것 |
|---|---|
| 의료/진단 | 법무, 규제, 인증, 임상/검증, 개인정보 |
| SaaS | 기능 비교, 가격, 데모 신청, 온보딩, 보안 |
| 커머스 | 상품 상세, 리뷰, 구매 전환, 재구매, 배송 |
| B2B 제조 | 기술 사양, 인증, 유통망, 견적 문의, 파트너 |
| 교육 | 커리큘럼, 후기, 강사 신뢰, 상담 전환 |
| 로컬 비즈니스 | 지역 검색, 지도, 리뷰, 예약, 운영시간 |

### 성과
범용 구조를 유지하면서도 산업별 현실에 맞는 분석이 가능하다.

## 10. Final Rule

### 문제
템플릿이 있어도 운영 원칙이 없으면 다시 산출물 중심으로 흐를 수 있다.

### 원인
브랜드 성장 프로젝트는 분석, 전략, 실행, 검수가 계속 연결되어야 하기 때문이다.

### 해결
아래 원칙을 항상 유지한다.

### 실행
1. 실행보다 분석을 먼저 한다.
2. IA를 먼저 만든다.
3. 바로 디자인하지 않는다.
4. 항상 Why부터 설명한다.
5. 항상 경쟁사와 비교한다.
6. SEO와 GEO를 동시에 고려한다.
7. 모든 결과물은 문서화한다.
8. 항상 다음 Action을 제안한다.
9. 모든 Deliverable은 재사용 가능하게 만든다.
10. 프로젝트 전체 일관성을 유지한다.

### 성과
다른 사업분야에서도 같은 품질의 브랜드 성장 프로젝트를 운영할 수 있다.
