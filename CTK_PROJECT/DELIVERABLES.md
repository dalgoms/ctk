# CTK ChoiceDX Deliverables

## Deliverables 관리 기준

### 문제
- 3개월 동안 만들어야 할 산출물이 많아 목적, 담당, 완료조건, 일정이 불명확해질 수 있다.

### 원인
- 경쟁사 분석, SEO/GEO, 브랜드 포지셔닝, 홈페이지, 세일즈, PR, SNS, UX, Figma Review, Handover가 서로 연결되어 있기 때문이다.

### 해결
- 모든 Deliverable을 동일한 기준으로 관리한다.

### 실행
- 목적
- 담당
- 완료조건
- 예상 일정
- 필요한 자료
- 관련 Task

### 성과
- 산출물별 책임과 완료 기준이 명확해지고, 내부 담당자가 이어받기 쉬워진다.

## Deliverable Completion Checklist

### 문제
- 산출물이 만들어졌어도 실제 운영에 쓰이지 못하면 프로젝트 성과로 보기 어렵다.

### 원인
- 완료 기준이 작성 완료에만 머물고, 담당자 실행 가능성까지 확인하지 않을 수 있다.

### 해결
- 모든 Deliverable은 아래 기준을 통과해야 완료로 본다.

### 실행
- 목적이 명확한가
- 담당자가 정해져 있는가
- 완료조건이 확인 가능한가
- 필요한 자료 출처가 표시되어 있는가
- 관련 Task와 연결되어 있는가
- KPI 또는 검수 기준이 있는가
- 내부 담당자가 이어받을 수 있는가
- 다음 작업이 제안되어 있는가

### 성과
- Deliverable이 단순 문서가 아니라 실행 가능한 프로젝트 자산으로 남는다.
