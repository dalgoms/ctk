# Olive Young Reference Analysis

## Why
올리브영은 ChoiceDX의 직접 경쟁사가 아니다. 하지만 고객이 브랜드를 발견하고, 신뢰하고, 혜택을 이해하고, 구매 또는 문의로 이동하는 구조를 가장 잘 보여주는 국내 레퍼런스다.

## What
웹스카우트로 수집한 올리브영 URL을 기준으로 IA, SEO/GEO, Trust, Channel, Conversion 관점에서 참고 가능한 구조를 분류한다.

## How
URL을 기능별로 나누고, ChoiceDX에 적용 가능한 시사점만 추출한다.

## Result
ChoiceDX는 올리브영의 커머스 구조를 그대로 따라가는 것이 아니라, “신뢰를 먼저 만들고, 고객 행동을 짧게 연결하는 구조”를 참고해야 한다.

---

## 1. URL 분류

### 문제
수집된 URL에는 올리브영 핵심 페이지뿐 아니라 브라우저 다운로드 링크, CJ 계열사 링크, SNS 링크가 섞여 있다.

### 원인
웹스카우트가 페이지 내 외부 링크까지 함께 수집했기 때문이다.

### 해결
분석에 사용할 URL과 제외할 URL을 먼저 분류한다.

### 실행
| 분류 | URL 유형 | 분석 활용 여부 | 이유 |
|---|---|---|---|
| Browser Download | Internet Explorer, Chrome, Firefox, Opera | 제외 | 올리브영 IA/브랜드 분석과 직접 관련 없음 |
| Commerce Core | 메인, 랭킹, 세일, 기획전, 이벤트, 핫딜 | 활용 | 고객 탐색과 전환 흐름 파악 가능 |
| Customer Service | 주문배송, FAQ, 매장안내 | 활용 | 신뢰와 고객 불안 해소 구조 파악 가능 |
| Brand / Corporate | 브랜드 스토리, 기업 사이트, 채용 | 활용 | 브랜드 신뢰, 조직 신뢰, Entity 강화 참고 가능 |
| Partner / B2B | 입점/파트너 관련 URL | 활용 | 파트너 신뢰와 B2B 전환 구조 참고 가능 |
| Global | 글로벌 올리브영 | 활용 | 글로벌 노출, 다국어, 해외 고객 접점 참고 가능 |
| CJ Group Network | CJ 계열사 링크 | 제한 활용 | 그룹 신뢰와 외부 권위 구조 참고 |
| SNS | Facebook, Instagram, YouTube | 활용 | 브랜드 Entity와 외부 Citation 구조 참고 가능 |

### 성과
올리브영 URL은 경쟁 분석이 아니라 “대형 브랜드의 신뢰/탐색/전환 구조 레퍼런스”로 사용할 수 있다.

## 2. Reference IA

### 문제
ChoiceDX는 고객이 제품을 이해하고 문의하기까지 필요한 정보 구조가 아직 충분히 검증되지 않았다.

### 원인
현재는 제품, 기술, 신뢰, FAQ, 문의, 글로벌 정보가 어떤 순서로 연결되어야 하는지 추가 판단이 필요하다.

### 해결
올리브영의 IA를 참고해 ChoiceDX에 맞는 B2B/진단형 IA로 변환한다.

### 실행
| Olive Young 구조 | 역할 | ChoiceDX 적용 방향 |
|---|---|---|
| 메인 | 전체 브랜드 진입점 | ChoiceDX가 무엇이고 누구를 위한 솔루션인지 첫 화면에서 설명 |
| 랭킹 / 베스트 | 선택 기준 제공 | 인기/검증/활용 사례 대신 “대표 사용 시나리오” 제공 |
| 세일 / 이벤트 / 핫딜 | 행동 유도 | 프로모션보다 상담/도입 문의 CTA로 변환 |
| 기획전 / 테마관 | 고객 관심사별 큐레이션 | 피부/두피/건강 데이터 등 문제 기반 콘텐츠 허브로 변환 |
| FAQ | 구매 전 불안 해소 | 진단 정확도, 데이터, 도입 방식, 법무 표현 관련 FAQ 구성 |
| 매장안내 | 오프라인 접점 신뢰 | 파트너/도입처/운영 프로세스 신뢰 요소로 변환 |
| 브랜드 스토리 | 브랜드 철학과 신뢰 | CTK/ChoiceDX의 기술 배경, 목적, 고객 가치 설명 |
| 글로벌 사이트 | 해외 접점 | 영문 Entity, 글로벌 SEO/GEO, 해외 파트너 대응 구조 참고 |

### 성과
ChoiceDX IA는 “제품 소개” 중심이 아니라 “고객 문제 → 기술 이해 → 신뢰 확인 → 문의” 흐름으로 설계해야 한다.

## 3. SEO / GEO Implications

### 문제
검색엔진과 AI Search는 브랜드를 단일 페이지가 아니라 여러 출처의 연결된 Entity로 이해한다.

### 원인
공식 사이트, 기업 사이트, SNS, 파트너 페이지, 외부 계열 링크가 함께 브랜드 권위를 만든다.

### 해결
ChoiceDX도 공식 페이지와 외부 Citation을 함께 설계한다.

### 실행
| 항목 | Olive Young 관찰 | ChoiceDX 적용 |
|---|---|---|
| Brand Entity | 공식몰, 기업 사이트, 브랜드 스토리, SNS가 연결됨 | CTK, ChoiceDX, 제품 카테고리, 기술 설명을 일관된 Entity로 정리 |
| Discoverability | 메인, 이벤트, 랭킹, FAQ, 매장, 글로벌 URL이 검색 접점을 넓힘 | 제품 페이지, FAQ, 기술 설명, 사례, 문의, 글로벌 페이지 구성 |
| Authority | CJ 계열/기업/채용/파트너 링크가 신뢰를 보강 | CTK 회사 정보, 파트너, 레퍼런스, 인증, PR 출처 연결 |
| AI Search | 여러 공식/외부 출처가 AI 답변의 근거가 될 수 있음 | AI가 인용 가능한 정의 문장, FAQ, 구조화 데이터, 외부 언급 확보 |
| Language | 국내/글로벌 접점이 분리됨 | 국문/영문 페이지 역할을 분리하고 동일한 Entity로 연결 |

### 성과
ChoiceDX는 SEO만이 아니라 GEO 관점에서 “AI가 설명하기 쉬운 브랜드 구조”를 만들어야 한다.

## 4. Trust Structure

### 문제
진단/기술 기반 브랜드는 고객이 제품을 이해하기 전에 신뢰를 먼저 확인하려 한다.

### 원인
정확도, 데이터, 법무 표현, 도입 효과, 운영 방식에 대한 불안이 크기 때문이다.

### 해결
올리브영의 신뢰 접점을 ChoiceDX에 맞는 B2B 신뢰 구조로 변환한다.

### 실행
| Trust Element | Olive Young Reference | ChoiceDX 적용 |
|---|---|---|
| Corporate Trust | 기업 사이트, CJ 네트워크 | CTK 회사 소개, 기술 배경, 사업 신뢰 |
| Customer Assurance | FAQ, 주문배송, 고객센터 | 도입 FAQ, 데이터 처리, 진단 프로세스, 문의 안내 |
| Offline / Partner Trust | 매장안내, 파트너 링크 | 도입처, 파트너, 운영 환경, 영업자료 |
| Social Proof | SNS, YouTube, Instagram | PR, 고객 사례, 데모 영상, 세일즈 자료 |
| Global Trust | 글로벌 사이트 | 영문 소개, 글로벌 적용 가능성, 해외 검색 대응 |

### 성과
ChoiceDX 홈페이지에는 “기술 설명”뿐 아니라 “안심하고 문의할 수 있는 근거”가 반드시 포함되어야 한다.

## 5. ChoiceDX 적용 전략

### 문제
ChoiceDX가 올리브영처럼 대형 커머스 구조를 그대로 따라가면 제품 특성과 맞지 않는다.

### 원인
ChoiceDX는 구매형 커머스가 아니라 B2B/진단/기술 기반 브랜드이기 때문이다.

### 해결
올리브영의 원리를 변환해서 적용한다.

### 실행
| 전략 | 설명 | Expected Output |
|---|---|---|
| Product Page as Entity Hub | 제품 페이지를 AI Search가 이해할 수 있는 대표 출처로 만든다 | Product / Technology / FAQ / Schema |
| FAQ as Trust Layer | 고객 불안을 FAQ로 선제 해소한다 | Diagnosis FAQ, Data FAQ, Legal-safe FAQ |
| Corporate Trust Link | CTK 회사 신뢰와 ChoiceDX 제품 신뢰를 연결한다 | About CTK, Brand Story, PR Link |
| Problem-Based Content Hub | 고객 문제별 진입 페이지를 만든다 | Skin / Scalp / Wellness / Retail Use Case |
| Global Search Layer | 국문과 영문 검색 모두 대비한다 | Korean Page, English Summary, Global Metadata |
| Partner Conversion Flow | 문의 CTA를 단순 버튼이 아니라 도입 흐름으로 만든다 | Contact Flow, Sales Kit, Partner Guide |

### 성과
ChoiceDX는 “작지만 신뢰도 높은 IA”를 만들 수 있다. 올리브영의 강점은 규모가 아니라, 고객 행동을 분명하게 연결하는 구조에서 가져와야 한다.
